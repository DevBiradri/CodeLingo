# CodeLingo Methodology

## Content Hierarchy
1. **Course:** (e.g., Intro to AI)
2. **Unit:** (e.g., Python for ML)
3. **Skill:** (e.g., Variables & Data Types)
4. **Lesson:** A series of 5-10 challenges.
5. **Challenge:** The individual interactive task.

## Challenge Types
- **Parsons Problems:** Drag-and-drop code blocks to build a valid snippet.
- **Fill-in-the-blank:** Complete code by typing or selecting missing syntax (e.g., `for i in _____(5):`).
- **Output Predictor:** "What does this code print?" with multiple choice or text input.
- **Syntax Matcher:** Match terms (e.g., "Integer") with examples (e.g., "42").
- **Tiny Sandbox:** A minimal code editor for writing 1-3 lines of code.

## Gamification Mechanics
- **Hearts (Lives):** Start with 5. Losing all hearts locks lessons for X hours or requires "refill" via practice.
- **Streaks:** Daily activity tracker with fire icons.
- **XP (Experience Points):** Earned per challenge and bonus for perfect lessons.
- **Boss Levels:** Longer, cumulative lessons at the end of each Unit.
- **Lore Selection:** AI-driven narrative skin applied to all UI text and assets.
